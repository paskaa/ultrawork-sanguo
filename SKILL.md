## 模型说明

### Qoder/OpenCode 平台支持的模型

| 模型 | 费用 | 特点 | 适用场景 |
|------|------|------|----------|
| `qwen3.5-plus` | 0.2x | 阿里最新，编程强 | 开发任务 |
| `glm-5` | 0.5x | 智谱旗舰，系统工程 | 调度/战略 |
| `kimi-k2.5` | 0.3x | 月之暗面，多模态 | 复杂任务 |
| `minimax-m2.5` | 0.2x | MiniMax，快性价比 | 探索/分析 |

### 费用对比

```
之前: 4 × ultimate (1.6x) = 6.4x
现在: 
  - 主帅调度: glm-5 (0.5x)
  - 中层3将: 0.5 + 0.2 + 0.2 = 0.9x
  - 底层6将: 0.2×4 + 0.3×2 = 1.4x
  总计: 2.8x (节省56%)
```

## 并行执行规则

**必须在同一消息中发送多个Task调用！**

```javascript
// ✅ 正确：并行执行
// 在一个回复中同时调用多个Agent
Task({subagent_type: "zhaoyun", ...})
Task({subagent_type: "simayi", ...})

// ❌ 错误：串行执行
await Task(...)  // 等待完成
await Task(...)  // 再执行下一个
```

## 任务调度策略

### 任务分类

| 类别 | 关键词 | 主将 | 副将 |
|------|--------|------|------|
| 前端开发 | UI, 界面, 样式, Vue | zhaoyun | gaoshun |
| 后端开发 | API, 接口, 数据库 | zhaoyun | chendao |
| 架构设计 | 设计, 方案, 架构 | zhouyu | lusu |
| 代码探索 | 搜索, 查找, 分析 | simayi | simashi |
| 快速修复 | bug, 修复, fix | zhaoyun | - |

### 调度流程

```
1. ZhugeLiang 分析意图
2. 根据任务类型选择主将
3. 主将调用下属部将
4. 并行执行提高效率
5. 汇总结果输出
```
