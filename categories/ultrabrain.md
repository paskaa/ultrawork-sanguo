# Ultrabrain - 复杂决策类别

## 类别描述

复杂逻辑、架构决策、战略规划。涉及系统设计、技术选型、重大决策等需要深度思考和全面分析的任务。

## 典型任务

### 高匹配度
- "设计支付系统架构"
- "制定微服务拆分方案"
- "评估技术选型"
- "规划系统演进路线"

### 中等匹配度
- "分析性能瓶颈"
- "制定优化策略"
- "设计数据模型"

### 低匹配度
- "实现登录功能"（属于 deep）
- "修复 bug"（属于 quick）

## 执行策略

### 分配 Agent
- **Prometheus**: 主要执行者（访谈式规划）
- **Explorer**: 信息收集支持

### 执行原则
1. **先问后做** - 澄清需求再规划
2. **多方案对比** - 提供多个可选方案
3. **全面考虑** - 技术、业务、成本等多维度
4. **可执行** - 规划要能落地

### Prometheus 访谈模式
```
1. 需求澄清
   - 识别模糊点
   - 提出关键问题
   - 确认边界条件

2. 现状分析
   - 了解现有系统
   - 识别约束条件
   - 评估技术债务

3. 方案设计
   - 提出多个方案
   - 分析利弊
   - 给出推荐

4. 详细规划
   - 分解任务
   - 估算工作量
   - 识别风险
```

## 决策框架

### 技术选型决策

| 维度 | 评估项 |
|------|--------|
| 功能 | 是否满足需求 |
| 性能 | 是否满足性能要求 |
| 可维护性 | 社区活跃度、文档质量 |
| 成本 | 学习成本、运维成本 |
| 风险 | 技术风险、依赖风险 |

### 架构设计决策

```
评估维度:
├── 功能性
│   ├── 需求覆盖
│   ├── 扩展性
│   └── 灵活性
├── 非功能性
│   ├── 性能
│   ├── 可靠性
│   ├── 安全性
│   └── 可观测性
├── 实施性
│   ├── 复杂度
│   ├── 工作量
│   └── 风险
└── 运维性
    ├── 部署复杂度
    ├── 监控难度
    └── 排障难度
```

## OpenHIS 架构约束

### 现有技术栈
- 后端: Java 17 + Spring Boot 2.5 + MyBatis Plus
- 前端: Vue 3 + Vite + Element Plus
- 数据库: PostgreSQL
- 缓存: Redis

### 必须遵循
- 三层架构: Web → Domain → Core
- 软删除模式: valid_flag
- 统一响应: AjaxResult
- 权限控制: @RequiresPermissions

### 扩展原则
- 优先使用现有技术栈
- 新技术引入需充分论证
- 保持向后兼容
- 渐进式演进
