# 代码重构工作流

6阶段完整流程，确保重构的安全性。

## 适用场景

- 代码重构
- 架构优化
- 性能优化
- 代码清理

## 触发方式

```
/workflow refactor <重构描述>
```

## 6阶段流程

### Phase 1: 现状分析 (Analysis)

**目标**: 分析代码现状，识别重构点

**执行将领**:
- **司马懿** - 代码探索
- **司马师** - 深度分析

**输出**: 重构分析报告

---

### Phase 2: 重构规划 (Planning)

**目标**: 制定重构计划

**执行将领**:
- **周瑜** - 重构规划
- **鲁肃** - 风险评估

**输出**: 重构计划、回滚方案

---

### Phase 3: 重构执行 (Refactor)

**目标**: 执行代码重构

**执行将领**:
- **赵云** - 主重构
- **高顺** - 前端重构
- **陈到** - 后端重构

**输出**: 重构后的代码

---

### Phase 4: 影响审查 (Review)

**目标**: 审查重构影响

**执行将领**:
- **关羽** - 代码审查
- **关平** - 规范检查
- **周仓** - 安全检查

**输出**: 审查报告

---

### Phase 5: 回归测试 (Test)

**目标**: 确保重构不破坏功能

**执行将领**:
- **徐庶** - 测试协调
- **庞林** - 前端测试
- **严颜** - 后端测试

**输出**: 测试报告

---

### Phase 6: 性能监控 (Monitor)

**目标**: 监控重构效果

**执行将领**:
- **满宠** - 综合监控
- **程昱** - 前端监控
- **贾诩** - 后端监控

**输出**: 监控报告

---

## 验收标准

1. 功能保持一致
2. 性能不下降
3. 代码质量提升
4. 测试全部通过

**不通过**: 返回 Phase 3 重新重构，最多重试3次。
