# 功能开发工作流

6阶段完整流程，确保功能开发的质量。

## 适用场景

- 新功能开发
- 模块实现
- 页面开发
- API开发

## 触发方式

```
/workflow feature <功能描述>
```

## 6阶段流程

### Phase 1: 需求分析 (Requirement)

**目标**: 澄清需求，明确范围

**执行将领**:
- **周瑜** - 需求访谈
- **鲁肃** - 需求分析

**输出**: 需求文档、功能清单

---

### Phase 2: 架构设计 (Architecture)

**目标**: 设计技术方案

**执行将领**:
- **周瑜** - 架构设计
- **鲁肃** - 方案评估

**输出**: 技术方案、接口设计

---

### Phase 3: 开发实现 (Develop)

**目标**: 实现功能代码

**执行将领**:
- **赵云** - 主开发
- **高顺** - 前端开发
- **陈到** - 后端开发

**输出**: 功能代码、API接口

---

### Phase 4: 代码审查 (Review)

**目标**: 确保代码质量

**执行将领**:
- **关羽** - 代码审查
- **关平** - 规范检查

**输出**: 审查报告

---

### Phase 5: 测试验证 (Test)

**目标**: 验证功能正确性

**执行将领**:
- **徐庶** - 测试协调
- **庞林** - 前端测试
- **严颜** - 后端测试
- **刘晔** - E2E测试

**输出**: 测试报告

---

### Phase 6: 部署监控 (Deploy)

**目标**: 上线并监控

**执行将领**:
- **满宠** - 部署监控
- **程昱** - 前端监控
- **贾诩** - 后端监控

**输出**: 部署报告、监控数据

---

## 验收标准

1. 功能完整实现
2. 代码审查通过
3. 测试覆盖率达到80%
4. 监控无异常

**不通过**: 返回 Phase 3 重新开发，最多重试3次。
