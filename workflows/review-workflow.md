# 代码审查工作流

多维度代码质量检查流程。

## 适用场景

- 代码审查
- 质量评估
- 安全检查
- 规范检查

## 触发方式

```
/workflow review <审查范围>
```

## 4阶段流程

### Phase 1: 静态分析 (Static)

**目标**: 静态代码分析

**执行将领**:
- **司马懿** - 代码探索
- **司马师** - 深度分析

**输出**: 静态分析报告

---

### Phase 2: 安全检查 (Security)

**目标**: 安全漏洞扫描

**执行将领**:
- **关羽** - 安全审查
- **周仓** - 漏洞扫描

**输出**: 安全报告

---

### Phase 3: 质量评估 (Quality)

**目标**: 代码质量评估

**执行将领**:
- **关平** - 规范检查
- **赵云** - 代码审查

**输出**: 质量报告

---

### Phase 4: 综合报告 (Report)

**目标**: 生成审查报告

**执行将领**:
- **司马昭** - 信息整理
- **诸葛亮** - 综合评估

**输出**: 完整审查报告

---

## 输出格式

```json
{
  "review": {
    "static": { "score": 85, "issues": [] },
    "security": { "score": 90, "vulnerabilities": [] },
    "quality": { "score": 88, "suggestions": [] },
    "overall": { "score": 87, "status": "PASS" }
  }
}
```
