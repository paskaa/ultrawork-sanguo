# 禅道任务处理工作流

从禅道系统自动获取任务信息，分析任务类型，并行调动三国军团各团队执行。

## 快速使用

### 处理禅道任务
```bash
/zt https://zentao.gentronhealth.com/index.php?m=bug&f=view&id=123
```

### 处理需求
```bash
/zt https://zentao.gentronhealth.com/index.php?m=story&f=view&id=456
```

### 处理任务
```bash
/zt https://zentao.gentronhealth.com/index.php?m=task&f=view&id=789
```

## 团队协作矩阵

### 情报团队 (司马懿 + 司马师)
| 职责 | 工具 | 输出 |
|------|------|------|
| 代码定位 | grep, glob | 相关文件列表 |
| 架构分析 | read | 模块关系图 |
| 依赖追踪 | grep | 依赖关系 |

### 监控团队 (满宠 + 程昱 + 贾诩)
| 职责 | 范围 | 输出 |
|------|------|------|
| 前端监控 | Console/Network/Vue | 前端错误报告 |
| 后端监控 | API/DB/System | 后端异常报告 |
| 综合诊断 | 日志分析 | 问题根因 |

### 执行团队 (根据任务类型)
| 任务类型 | 主将 | 部将 | 模型 |
|----------|------|------|------|
| BUG修复 | 张飞 | 雷绪、吴兰 | MiniMax-M2.5 |
| 前端开发 | 赵云 | 高顺 | Qwen3-Coder |
| 后端开发 | 赵云 | 陈到 | Qwen3-Coder |
| 架构设计 | 周瑜 | 鲁肃、黄盖 | GLM-5 |
| 复杂功能 | 赵云 | 高顺、陈到 | Qwen3-Coder |

### 质量团队 (关羽 + 关平 + 周仓)
| 职责 | 检查项 | 输出 |
|------|--------|------|
| 代码风格 | 规范、最佳实践 | 风格问题列表 |
| 安全检查 | 漏洞、风险 | 安全报告 |
| 逻辑审查 | 边界、异常 | 审查意见 |

## 任务路由规则

### BUG 类型路由
```yaml

  1. 雷绪快速定位问题代码
  2. 张飞分析并修复
```

```yaml

  4. 关羽审查代码
```

## 并行执行策略

### 策略1: 情报先行
```
时间线:
T0: 司马懿开始代码定位
T1: 司马懿输出文件列表
T2: 主执行者开始修改 (基于司马懿的定位结果)
```

### 策略2: 监控并行
```
时间线:
T0: 程昱检查前端日志
T0: 贾诩检查后端日志  (并行)
T1: 满宠汇总监控报告
```

### 策略3: 质量后置
```
时间线:
T0-T3: 执行团队修改代码
T4: 关羽团队审查代码 (代码完成后)
```

## 禅道URL解析

### 支持的URL格式
| 类型 | URL格式 | 解析 |
|------|---------|------|
| BUG | `/m=bug&f=view&id={id}` | 提取BUG详情 |
| 需求 | `/m=story&f=view&id={id}` | 提取需求详情 |
| 任务 | `/m=task&f=view&id={id}` | 提取任务详情 |
| 文件 | `/m=file&f=read&fileID={id}` | 下载附件图片 |

### 页面解析规则
```javascript
// 提取任务信息
const taskInfo = {
  type: extractType(url),        // bug/story/task
  id: extractId(url),            // 任务ID
  title: extractTitle(content),  // 标题
  description: extractDesc(content), // 描述
  priority: extractPriority(content), // 优先级
  screenshots: extractImages(content) // 截图
};
```

## 配置

### 禅道系统配置
```yaml
zentao:
  url: https://zentao.gentronhealth.com
  auth:
    type: cookie  # 或 token
    # 登录凭证在环境变量中配置
```

### 环境变量
```
ZENTAO_URL=https://zentao.gentronhealth.com
ZENTAO_USERNAME=xxx
ZENTAO_PASSWORD=xxx
```

## 注意事项

1. **权限验证**: 确保有禅道系统访问权限
2. **并行限制**: 最多同时运行 5 个子任务
3. **超时处理**: 单个任务超过 10 分钟需要人工介入
4. **结果确认**: 重要修改需要用户确认后再提交
