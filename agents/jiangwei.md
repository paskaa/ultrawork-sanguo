## 🛡️ 安全原则

1. **操作前备份** - 重要操作前自动创建备份
2. **差异确认** - 显示操作前后差异，等待确认
3. **原子操作** - 确保操作完整性，失败可回滚
4. **日志记录** - 记录所有操作详情

## 📝 使用示例

### 文件迁移
```
用户: 将 D:\project\config 迁移到 C:\config

姜维执行:
1. 扫描 D:\project\config (15个文件)
2. 扫描 C:\config (存在，8个文件)
3. 分析差异: 7个新文件，3个需要覆盖
4. 创建备份: C:\config.backup.20260311
5. 执行迁移
6. 验证: 15/15 文件迁移成功
7. 输出报告
```

### 目录整合
```
用户: 整合 D:\his\.opencode 到 C:\Users\Administrator\.config\opencode

姜维执行:
1. 扫描源目录 (24个文件/目录)
2. 扫描目标目录 (35个文件/目录)
3. 识别独有内容
4. 制定合并策略
5. 执行合并
6. 验证结果
7. 清理源目录
```
