# 禅道任务处理工作流

你收到一个禅道任务，需要按照以下工作流程处理：

## 任务URL

$ARGUMENTS

## 工作流程

### 阶段1: 信息获取 (司马懿团队 - 情报侦察)

1. **访问禅道URL** - 使用 webfetch 工具获取禅道页面内容
2. **解析任务信息**:
   - 任务类型: BUG / 需求 / 任务
   - 任务标题
   - 任务描述
   - 优先级
   - 相关截图/附件
3. **提取关键信息**:
   - 错误信息/异常堆栈
   - 涉及的模块/页面
   - 复现步骤
   - 预期结果 vs 实际结果

### 阶段2: 任务分类与路由

根据任务类型，选择对应的主将:

| 任务类型 | 主将 | 部将 | 类别 |
|----------|------|------|------|
| BUG修复 | 张飞 | 雷绪、吴兰 | quick |
| 功能开发 | 赵云 | 高顺、陈到 | deep |
| 前端任务 | 赵云 | 高顺 | visual-engineering |
| 后端任务 | 赵云 | 陈到 | deep |
| 架构设计 | 周瑜 | 鲁肃、黄盖 | ultrabrain |
| 代码审查 | 关羽 | 关平、周仓 | review |
| 监控诊断 | 满宠 | 程昱、贾诩 | monitor |

### 阶段4: 执行步骤

1. **定位问题** - 调用 @simayi 搜索相关代码文件
2. **诊断分析** - 调用 @manchong 监控日志异常
3. **制定方案** - 复杂问题调用 @zhouyu 规划
4. **执行修复/开发** - 调用对应主将执行
5. **质量检查** - 调用 @guanyu 审查代码
6. **结果汇总** - 整合所有结果输出
