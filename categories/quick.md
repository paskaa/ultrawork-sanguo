# Quick - 快速修复类别

## 类别描述

单文件修改、小修复、快速调整。涉及 Bug 修复、小改动、配置更新等简单快速的任务。

## 典型任务

### 高匹配度
- "修复登录失败的问题"
- "修改用户列表显示错误"
- "修复表单验证 bug"
- "更新配置文件"

### 中等匹配度
- "调整表格列宽"
- "修改提示文字"
- "更新下拉选项"

### 低匹配度

## 执行策略

### 分配 Agent
- **Hephaestus**: 唯一执行者

### 执行原则
1. **快速定位** - 使用 Explorer 能力快速找到问题
2. **精准修复** - 只改需要改的地方
3. **最小影响** - 不引入不必要的变更
4. **验证修复** - 确认问题已解决

### 执行流程
```
1. 定位问题
   - 搜索相关代码
   - 分析错误日志
   - 理解问题原因

2. 实施修复
   - 最小化修改
   - 遵循现有规范
   - 不重构周围代码

3. 验证修复
   - 确认问题解决
   - 检查无副作用
   - 简单测试
```

## Bug 修复模式

### 常见 Bug 类型

| 类型 | 定位方法 | 修复策略 |
|------|----------|----------|
| 空指针 | 搜索访问点 | 添加判空 |
| 类型错误 | 检查数据流 | 类型转换/校验 |
| 逻辑错误 | 跟踪执行路径 | 修正条件 |
| 并发问题 | 分析共享状态 | 加锁/同步 |
| SQL 错误 | 检查 SQL 语句 | 修正语法 |

### 修复检查清单

- [ ] 确认问题根因
- [ ] 最小化修改范围
- [ ] 不引入新问题
- [ ] 添加必要注释（如果逻辑不直观）
- [ ] 验证修复有效

## OpenHIS 常见问题

### 前端问题
```javascript
// 问题: 表单验证不生效
// 定位: 检查 rules 和 prop 是否匹配
// 修复: 确保 prop 名称与 v-model 绑定的字段一致

// 问题: 接口返回 401
// 定位: 检查 token 和权限配置
// 修复: 添加权限注解或配置白名单
```

### 后端问题
```java
// 问题: 查询返回空
// 定位: 检查 valid_flag 和查询条件
// 修复: 添加 .eq("valid_flag", 1)

// 问题: 事务不生效
// 定位: 检查 @Transactional 注解
// 修复: 确保方法为 public，检查异常处理
```

### 数据库问题
```sql
-- 问题: 查询慢
-- 定位: 检查执行计划
-- 修复: 添加索引

-- 问题: 中文乱码
-- 定位: 检查编码配置
-- 修复: 确保 UTF-8 编码
```
