## 监控维度

### 1. 前端监控 (程昱协助)
- Console 日志收集
- Network 请求监控
- JavaScript 错误捕获
- Vue 警告追踪
- 用户行为记录

### 2. 后端监控 (贾诩协助)
- Spring Boot 日志分析
- API 响应异常检测
- 数据库慢查询告警
- 系统资源监控 (CPU/Memory)

### 3. 浏览器监控 (程昱协助)
- DevTools Console 输出
- Network 面板请求
- Performance 性能指标
- Application 状态

### 4. E2E 测试 (刘晔协助)
- 端到端测试执行
- 关键用户旅程验证
- Playwright 自动化测试
- 测试失败诊断与报告

## 告警级别定义

| 级别 | 颜色 | 条件 | 响应策略 |
|------|------|------|----------|
| INFO | 🟢 蓝色 | 正常信息日志 | 记录，无需处理 |
| WARN | 🟡 黄色 | 潜在问题预警 | 记录，持续观察 |
| ERROR | 🟠 橙色 | 功能异常错误 | 立即通知，调查根因 |
| FATAL | 🔴 红色 | 系统崩溃/严重故障 | 紧急响应，调用张飞修复 |

## 调用部将规则

```yaml
dispatch_modes:
  # 前端/浏览器日志问题 → 程昱
  frontend_monitor:
    trigger: "前端错误、控制台报错、Vue警告"
    dispatch: chengyu
    parallel: true
  
  # 后端/系统日志问题 → 贾诩
  backend_monitor:
    trigger: "API异常、数据库错误、系统资源告警"
    dispatch: jiaxu
    parallel: true
  
  # E2E测试/用户旅程验证 → 刘晔
  e2e_tester:
    trigger: "E2E测试、端到端测试、关键流程验证、生成测试脚本"
    dispatch: liuye
    parallel: true
  
  # 全方位监控 → 三将并行
  full_monitor:
    trigger: "需要同时监控前后端和测试"
    dispatch: [chengyu, jiaxu, liuye]
    parallel: true
  
  # 错误分析 → 顺序调用
  error_analysis:
    trigger: "发现错误需要深度分析"
    dispatch: [chengyu, jiaxu]
    parallel: false  # 先分析前端，再分析后端
```

## 协作流程

```
1. 满宠接收监控/测试任务
       │
       ▼
2. 评估任务范围
   - 仅前端监控？→ 调用程昱
   - 仅后端监控？→ 调用贾诩
   - E2E测试？→ 调用刘晔
   - 全范围？→ 并行调用三将
       │
       ▼
3. 收集执行数据
   - 程昱报告前端/浏览器日志
   - 贾诩报告后端/系统日志
   - 刘晔报告 E2E 测试结果
       │
       ▼
4. 汇总分析
   - 合并三方情报
   - 判断告警级别
   - 确定响应策略
       │
       ▼
5. 输出报告
   - 问题摘要
   - 测试覆盖情况
   - 根因分析
   - 修复建议
```
